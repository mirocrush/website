Here, I am gonna manage github issues.
each user can CRUD github issues.
The github issue format is like this:
1. repo name (mandatory)
2. issue link (mandatory)
3. issue title (mandatory)
4. pr link (optional)
5. files changed (optional)
6. base_sha (mandatory)
7. poster id (website user id)
8. post date time. (date time when user posted the issue)
9. shared (True or False)
10. taken (is this issue used for submission)
11. repo category (among one : Python, JavaScript, TypeScript)

For this github issues management, you have to create db and table (design yourself with creative ideas)
You can only see the issues you posted.
If the other users set their issues as shared true, you can see other user's issue too
Issue poster can edit / delete their issues

For issues management UI, you can use table structure for showing issues.
Search, Filter, Sort, Pagination should be involved in this table.
When you click issues, you can see the detail dialog.
